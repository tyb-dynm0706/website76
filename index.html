GitHub Pagesメモ

新規websiteの作り方

ページをつくるための初期設定編

  1. Home ＞ 「NEW」 をクリック
  2. General ＞ Repository nameにフォルダ名となるものを入力（ここではwebsite76と入力した）
  3. Configuration ＞ 「Choose visibility」 ＞ 「Public（公開）」にチェック。
    ＊privateでも後ほど変更できる。
  4. 「Create repository」ボタンを押す

HTMLファイルを作る編

  1. 青枠の中にある「creating a new file」というテキストをクリック
  2. 上部のファイル名に「index.html」を入力
    例　website76/「index.html」in main
  3. htmlを書く
  4. 右上にある緑色の「Commit changes...」ボタンを押す
  5. 「Commit changes」ボタンを押す

公開設定編

  1. 「Settings」タブを開く
  2. 左メニューにある「Pages」を選択
  3. 「Build and deployment」 ＞ 「Branch」 ＞ 「None」→「main」変え「Save」ボタンを押す
  4. 右上にある緑色の「Commit changes...」ボタンを押す
  5. しばらく待つ（少し時間がかかった。deploymentが成功するとメール届く）
  6. 処理が終わると「Build and deployment」の上の「GitHub Pages」の項目に作成したURLを確認。
    Your site is live at 「https://（GitHubユーザー名）.github.io/（例 website76）/」のように表示される

ファイル編集編

  • 「リポジトリ名」を変更する
    「Settings」 ＞ 「Repository name」を変更し「Rename」ボタンを押して変更。
  • 「Private（非公開）」から「Public（公開）」に変更する
    「Settings」 ＞ 「General」 ＞ 「Danger Zone」 ＞ 「Change repository visibility」 ＞ 「Change visibility」 ＞ 「Change to public」 ＞ 「I want to make this repository public」ボタンを押す。
  • 編集しているページを簡単に確認するには、Google Chromeの拡張機能「GitHub HTML Preview」が優秀そう。すぐに導入できた。
  • ページを複数人で開発する方法
    「Settings」 ＞ 「collaborators」 ＞ 「Manage access」 ＞ 「add people」からメールを送信できる。

ブランチについて

main = 正式なバージョン
branch = テスト用
機能を追加する場合や大きく変更がある場合は、branchを作りそこでテストするイメージ。

  • ブランチの作り方
    「code」 ＞ 「Branches」 ＞ 「New branch」 ＞ 名前をつけて「Create new branch」ボタンを押して制作。
    Sourceは「main」にすると「main」から派生。
    制作した「branch」からも派生させることもできる。
